Technical FAQ

01
Smart ASR1

What does Smart ASR filter before the request reaches the brain?

keyboard_arrow_down

Our Smart ASR acts as an intelligent gatekeeper to keep the interaction model focused and efficient. It actively filters out "garbage" transcripts like random background noise, static, or tiny, meaningless character fragments that sometimes get falsely transcribed when you aren't actually speaking. It also performs smart echo rejection to ignore accidental microphone loop-backs. Only when it verifies that the audio contains genuine, intentional user speech does it forward the request to the primary brain for processing.

02
Primary Brain2

How does the 200ms context loop decide the next action?

keyboard_arrow_down

The orchestrator runs a high-speed control loop every 200 milliseconds that constantly monitors the entire state of your session. It evaluates whether you are still speaking, tracks the elapsed time of active timers, and checks the status of asynchronous background tasks. During every 200ms tick, it uses this information to decide exactly what to do next, including when to pass your request to the interaction model (Small Language Model) for processing.

03
Background Researcher3

When does the system delegate work to the background researcher?

keyboard_arrow_down

To maintain ultra-low latency, the interaction model handles basic chat, timer management, and activity tracking on its own. However, if a request requires heavy lifting like searching the web, connecting to MCP tools, or writing code to analyze data, it delegates the task to the background. This spins up the "Background Researcher" (e.g., Codex agent) to perform the complex, open-ended work asynchronously, allowing the main voice loop to remain fully responsive to you in the meantime.

04
Streaming TTS4

Which Text-to-Speech (TTS) providers are supported?

keyboard_arrow_down

The app provides fully configurable options for your Text-to-Speech output. You can easily plug in your preferred API provider based on your specific needs for latency and voice quality. Out of the box, we support providers like NVIDIA NIM, ElevenLabs, Sarvam, etc.
